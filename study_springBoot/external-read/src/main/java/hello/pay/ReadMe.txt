- @Profile -
프로필을 사용해서 각 환경마다 서로 다른 설정값을 적용하는 방법은 이전에 배웠다.
그런데 설정값이 아니라, 각 환경마다 서로 다른 '빈'을 등록해야 한다면 어떻게 해야 할까?
바로 이럴 때 필요한 것이 @Profile 애너테이션이다.
※@Profile 애너테이션의 정체 : https://inf.run/ANPrx(8분 39초 ~ 9분 48초) 참고

아래와 같은 상황이라 가정하고, 실습을 진행해 보자.

현재 나는 결제 기능을 구현해야 한다.
그런데 로컬 환경에서는 실제로 결제가 발생하면 안 되므로, 테스트 결제 기능이 있는 스프링 빈이 등록되도록 해야 하고,
운영 환경에서는 실제 결제 기능이 있는 스프링 빈이 등록되도록 해야 한다.
